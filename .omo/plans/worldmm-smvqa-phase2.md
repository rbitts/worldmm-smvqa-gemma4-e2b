### Must NOT have (guardrails, anti-slop, scope boundaries)
- NO datasets, model weights, checkpoints, or large eval artifacts copied to this host. Every pull has a size guard (fail and delete if the pulled set exceeds 20 MB).
- NO re-run or re-submission of the remote benchmark; phase 2 only reads existing artifacts under `$WORLDMM_OUTPUT_ROOT`.
- NO changes to metric/evaluation logic (`src/worldmm_smvqa/metrics.py`); official numbers stay exactly as produced remotely.
- NO Video-RAG / EgoButler reproduction lane (separate future task per HANDOFF caveat).
- NO secrets, tokens, or internal hostnames committed; docs use env-var placeholders (`$BASTION_HOST`, `$WORLDMM_OUTPUT_ROOT`, ...).

All ops todos are verified with agent-executed shell checks (`jq`, `rg`, `du`, exit codes). Existing gates (`uv run pytest`, `ruff check`, `basedpyright`) must stay green after any code/doc change.

- Note: "zero human intervention" applies to todo verification. The final-wave "wait for the user's explicit okay" is a post-completion review gate, outside agent verification.
