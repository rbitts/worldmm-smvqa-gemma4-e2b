# 논문 제목

| 항목 | 값 |
|---|---|
| Page ID | SM-PAPER-SLUG |
| Confluence parent | SM-PAPERS |
| 상태 | 검토 필요 / 검토 완료 / 재현 / 채택 / 기각 / 관찰 |
| 출판 | Peer-reviewed venue 또는 preprint 상태 |
| 1차 출처 | Official DOI, proceedings, arXiv link |
| 공식 code | Official repository, `미공개`, `미확인` |
| 최종 확인 | YYYY-MM-DD |
| 프로젝트 연결 | Claim ID, ADR, experiment |

## 핵심 결론

- **결정:** 채택 / 평가 / 보류 / 기각.
- **근거:** 이 프로젝트에 중요한 근거 또는 한계를 한 문장으로 적는다.
- **실행:** 연결할 claim, ADR, experiment 또는 명시적 보류 조건을 적는다.

## 근거 상태

`미재현`을 기본값으로 명시한다. 완료된 project experiment만 link하고,
paper-reported evidence와 local mock 값을 섞지 않는다.

## 논문 핵심

문제, 핵심 방법, 논문 결과를 3줄 이하로 요약한다.

## 근거

| Dataset | 조건 | Metric | 논문 보고 결과 | 위치 |
|---|---|---|---|---|
| 예시 | 예시 | 예시 | 예시 | Table 또는 figure |

1차 논문에서 검증된 값만 사용한다. 정확한 값이 확인되지 않은 경우 테이블을 제거한다.

## 판단 한계

- 논문에서 평가하지 않은 조건.
- 프로젝트 가설로 남아 있는 주장.

## 문제 배경

논문의 문제점, 입력 조건, 의도한 출력을 기술한다.

## 관련 방법

이 프로젝트와 관련된 구성 요소만 설명한다.
